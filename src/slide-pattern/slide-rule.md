##テンプレート一覧と使い分けルール
1. strong-message（強調メッセージ）
- 対応テンプレート：
「strong-message.md」の Marp テンプレート

- 役割：最重要メッセージをドーンと見せるスライド

# 主な使い所：

- 冒頭の問題提起・キーメッセージ

- セクション区切りの一言

- 最後のまとめ・一言で刺すメッセージ

- 注意：

図やスクショを見せたいときは使わない（後述テンプレートを優先）

- JSON形式：

json
コードをコピーする
{
  "type": "strong-message",
  "props": {
    "text": "黒板中央に表示する一文（できるだけ短くインパクトのある日本語）"
  }
}
2. list-3（黒板リスト・3行）
- 対応テンプレート：
「list-3.md」の Marp テンプレート

- 役割：3つの要素を並べて見せたいとき

#主な使い所：

- アジェンダ（3項目）

- 3つのポイント・3ステップ

- Before / After / Next のような三分構成

- JSON形式：

json
コードをコピーする
{
  "type": "list-3",
  "props": {
    "title": "スライドタイトル",
    "items": [
      "①のテキスト",
      "②のテキスト",
      "③のテキスト"
    ]
  }
}
3. list-4（4行リスト）
- 対応テンプレート：
「list-4.md」の Marp テンプレート

- 役割：箇条書きを4つ並べたいとき

#主な使い所：

- 課題・原因・特徴などが4つあるとき

- プロセスが4ステップのとき

- JSON形式：

json
コードをコピーする
{
  "type": "list-4",
  "props": {
    "title": "スライドタイトル",
    "items": [
      "①のテキスト",
      "②のテキスト",
      "③のテキスト",
      "④のテキスト"
    ]
  }
}
4. list-5（5行リスト）
- 対応テンプレート：
「list-5.md」の Marp テンプレート

- 役割：5つの要素をコンパクトに並べたいとき

- 主な使い所：

- チェックリスト

- ToDo・アクションプラン

- 条件・ルールなどが5個前後あるとき

- JSON形式：

json
コードをコピーする
{
  "type": "list-5",
  "props": {
    "title": "スライドタイトル",
    "items": [
      "①のテキスト",
      "②のテキスト",
      "③のテキスト",
      "④のテキスト",
      "⑤のテキスト"
    ]
  }
}
- メモ：
箇条書きの数に応じて list-3 / list-4 / list-5 を選び、6個以上になる場合は複数スライドに分割してください（例：list-3 + list-3）。

5. illustrations（イラスト画像つきスライド）
- 対応テンプレート：
「illustrations.md」の Marp テンプレート

- 役割：イラスト・概念図・アイコン系の画像で感覚的に伝えたいとき

#主な使い所：

- 抽象的なイメージ（理想の姿・未来像・世界観など）

- ストーリーの象徴的なシーン

- 感情的なニュアンスを伝えたいとき

- JSON形式：

json
コードをコピーする
{
  "type": "illustrations",
  "props": {
    "title": "スライドタイトル",
    "imagePrompt": "生成したいイラストの内容（日本語で具体的に）",
    "caption": "必要なら画像の補足説明（任意）"
  }
}
- imagePrompt は後続の画像生成でそのまま使えるよう、
「誰が・何をしていて・どんな雰囲気か」を具体的に書いてください。

6. screenshots（スクリーンショットパターン）
- 対応テンプレート：
「screenshots.md」の Marp テンプレート

- 役割：実際の画面・プロトタイプ・ツールのイメージを見せるスライド

#主な使い所：

- ツールやサービスのUI紹介

- 実際の操作手順・設定画面

- Before/Afterの比較（複数スライドで分けてもOK）

- JSON形式：

json
コードをコピーする
{
  "type": "screenshots",
  "props": {
    "title": "スライドタイトル",
    "target": "どの画面・どのサービスのスクリーンショットかの説明",
    "note": "どこを見てほしいか・ポイント（任意）"
  }
}
###スライド設計ルール（重要）
- 台本の順番を尊重する

- スライドの流れは、台本のストーリー順を崩さない範囲で組み立てる

ただし、同じメッセージの繰り返しはまとめてよい

#スライド粒度の目安

- 1つの強いメッセージ → strong-message

- 3〜5個の要素を並べたい → list-3 / list-4 / list-5

- 画面イメージが重要 → screenshots

- 感覚的・象徴的に見せたい → illustrations

#テキストの書き方

- 口語はスライド向けの短い文章に圧縮する

- 「〜していきたいと思っています」は「〜する」にするなど、簡潔に

- 主語・述語をはっきりさせる（誰が / 何を / どうするか）

- 枚数の目安

- 台本が長い場合も、意味のある塊ごとにスライド化し、不要に枚数を増やしすぎない

- 1つの話題が長い場合は「問題 → 解決策 → 効果」のように2〜3枚に分ける

#最終出力フォーマット
- 必ず 有効な JSON としてパース可能な形で出力すること

- ルートは {"slides":[ ... ]} のみ

- コメントや日本語の説明文は一切書かないこと
